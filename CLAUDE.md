## 🎯 Что за проект

PWA-помощник для настольной игры «Эрудит» (аналог Scrabble на русском языке). Виртуальная интерактивная доска 15×15, которая:
- фиксирует ходы (свои и соперника) в том же положении, что и на реальной доске;
- предлагает оптимальные ходы на основе словаря (Trie + anchor-алгоритм генерации);
- отслеживает "мешок" с буквами и прогнозирует их расход, включая джокеры соперника;
- считает очки с правильным аддитивным сложением словарных множителей (два ×3-слово → ×6, не ×9);
- предупреждает о риске "достройки" слова соперником (СЛОН → ЗАСЛОН → ЗАСЛОНКА);
- использует угловую эвристику (клетки А1/А15/П1/П15 — стратегически важны);
- поддерживает обмен букв и покупку джокеров;
- ведёт турнирную статистику по множителям х2/х3 для обоих игроков.

Целевые устройства: **iPad и Android-планшет в первую очередь** (тач-first UX, LetterPad вместо системной клавиатуры, нижняя таб-панель, safe-area для iOS standalone).

Полное ТЗ пользователя и детальный контекст решений — в `Chat_history.md` (там же обоснования выбора шрифтов, цветовой схемы и т.д.). Правила игры — в `erudit-description.md` и `erudit_boards.md`.

## 🛠 Технический стек

- **Vite + React 18 + TypeScript**, `npm create vite@latest . -- --template react-ts`
- **Tailwind CSS v4** + **shadcn/ui** (на базе `@base-ui/react`, НЕ Radix — разный API!)
- **Zustand** — стейт-менеджмент (`useGameStore.ts` + `selectors.ts`)
- **Web Workers** — загрузка словаря и генерация ходов в отдельном потоке (`game-engine.worker.ts`)
- **Vitest** — юнит-тесты для `game/`, `dictionary/`, `engine/` (без React/DOM зависимостей)
- **@fontsource** (self-hosted, для офлайн PWA): IBM Plex Sans, IBM Plex Sans Condensed, IBM Plex Mono — с кириллицей
- **vite-plugin-pwa** — манифест + service worker (ещё не подключен, см. Этап 9)
- **oxlint** — линтер
- Тёмная тема (единственная), палитра: графит-слива + медный акцент (действия/очки) + отдельный нефритовый акцент (только для статистики множителей)
- Деплой: GitHub Pages через `.github/workflows/deploy.yml` (ещё не создан, см. Этап 9)

## 🔑 Важные технические решения (не менять без причины)

- **Границы модулей:** `game/`, `dictionary/`, `engine/` — ноль импортов React, юнит-тестируемы и работают в Web Worker без DOM. Только `store/useGameStore.ts` вызывает эти модули; компоненты не обращаются к `game/*` напрямую, кроме форматирования для отображения.
- **Аддитивное сложение словарных множителей** — при пересечении слова несколькими клетками ×2/×3-слово множители складываются, затем применяются один раз (два ×3 → итог ×6, не ×9). Реализовано в `scoring.ts`, покрыто тестом — не регрессировать.
- **е/ё** — единственное место, знающее про фолдинг ё→е: `dictionary/normalize.ts`.
- **shadcn/ui на этом проекте использует `@base-ui/react`, а не Radix UI** — другой API, не путать при добавлении новых компонентов.
- **Тач-first UX** — превью хода по тапу (не только hover), тач-таргеты ≥44×44px, `LetterPad` вместо системной клавиатуры, нижняя таб-панель (не десктопный sidebar), safe-area padding для iOS standalone.
- `maximum-scale=1.0` в viewport meta был сознательно откачен — соображения доступности важнее фиксации зума.
- Словарь остаётся текущим (51k слов, `russian_words_nouns.txt`, е/ё уже свёрнуто) — решение пользователя, не менять на `nouns_with_yo.txt` без явного запроса.

Обновлять после завершения каждого этапа/подэтапа: сразу ставить ✅ и добавлять 1–2 строки комментария (что сделано, номер коммита). Не переписывать историю уже выполненных этапов — только дописывать.
